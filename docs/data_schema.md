# Data Schema Definition ŌĆö Personalized Networking Assistant

## Overview

Before building any service logic or routes, it is essential to define the data contracts between the frontend and backend. These contracts are expressed as **Pydantic models**, which provide:

- Automatic type validation
- Request/response serialization
- Clear API documentation through FastAPI's built-in Swagger UI

If a required field is missing or has the wrong type, FastAPI automatically returns a `422 Unprocessable Entity` error with a clear description ŌĆö without requiring any additional error-handling code.

All schema models are defined in `app/models/schemas.py`.

## Models

This file defines **six Pydantic model classes** that form the backbone of all data exchange in the application. Each class inherits from `BaseModel` to gain automatic validation.

---

### Input Models

#### `EventInput`
Represents a single event description submitted by the user.

```python
class EventInput(BaseModel):
    description: str
```

| Field | Type | Description |
|---|---|---|
| `description` | `str` | The event description text entered by the user |

---

#### `UserInterests`
Represents the list of interests provided by the user.

```python
class UserInterests(BaseModel):
    interests: List[str]
```

| Field | Type | Description |
|---|---|---|
| `interests` | `List[str]` | A list of interest keywords entered by the user |

---

#### `ConversationRequest`
Combined request model sent by the frontend when the user clicks **Generate Starters**. Contains both the event description and the user's interests.

```python
class ConversationRequest(BaseModel):
    description: str
    interests: List[str]
```

| Field | Type | Description |
|---|---|---|
| `description` | `str` | The event description text |
| `interests` | `List[str]` | List of user interest keywords |

---

#### `FactCheckRequest`
Request model for the fact-checking endpoint.

```python
class FactCheckRequest(BaseModel):
    query: str
```

| Field | Type | Description |
|---|---|---|
| `query` | `str` | The topic or claim the user wants to fact-check |

---

### Output Models

#### `ConversationResponse`
Response model returned after generating conversation starters. Contains the extracted themes and AI-generated suggestions.

```python
class ConversationResponse(BaseModel):
    topics: List[str]
    suggestions: List[str]
```

| Field | Type | Description |
|---|---|---|
| `topics` | `List[str]` | Themes extracted from the event description by DistilBERT |
| `suggestions` | `List[str]` | Conversation starters generated by GPT-2 |

---

#### `FactCheckResponse`
Response model returned after a Wikipedia fact-check query.

```python
class FactCheckResponse(BaseModel):
    summary: str
```

| Field | Type | Description |
|---|---|---|
| `summary` | `str` | Summarized result retrieved from the Wikipedia API |

## Why Pydantic?

| Feature | Benefit |
|---|---|
| Automatic validation | No manual type checking needed in route handlers |
| Clear error messages | FastAPI returns a `422` with field-level error details automatically |
| Swagger UI integration | All models appear as documented schemas in `/docs` |
| Serialization | Python objects are automatically converted to/from JSON |
